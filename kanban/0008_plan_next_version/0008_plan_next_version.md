**v1.1.1 以降の変更内容**:

**判定理由**:
1. ユーザー向けの新機能がない
2. バグ修正がない（最後のバグ修正は v1.0.0 → v1.1.0 時）
3. 変更がすべて内部インフラ・メタデータレベル
4. コア機能（テキスト変換ロジック）に変更なし
5. セマンティックバージョニング的に Patch リリースで十分

**Minor (1.2.0) に上げるべき場合**:
- ユーザー向けの新機能が追加された場合（例：新しいケース形式対応、新 CLI オプション）
